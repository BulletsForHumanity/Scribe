# Scribe

[![NuGet](https://img.shields.io/nuget/v/BulletsForHumanity.Scribe)](https://www.nuget.org/packages/BulletsForHumanity.Scribe)
[![Build & Test](https://github.com/BulletsForHumanity/Scribe/actions/workflows/build-test.yml/badge.svg)](https://github.com/BulletsForHumanity/Scribe/actions/workflows/build-test.yml)
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

A fluent DSL for writing Roslyn incremental source generators, code fix providers, and analyzers.

Scribe eliminates the painful `StringBuilder` / `IndentedTextWriter` boilerplate that comes with building Roslyn source generators. Instead of scattering raw string concatenation across your generator, you describe the output structure fluently and let the builder handle indentation, blank-line separation, using directives, namespaces, and XML documentation.

## Installation

```shell
dotnet add package BulletsForHumanity.Scribe
```

## Quick Start

```csharp
using Scribe;

var q = new Quill();
q.FileNamespace("MyApp.Generated");

q.Refs(
    "global::System.IParsable",
    "global::System.ArgumentException"
);

using (q.Block("public partial record struct Amount : global::System.IParsable<Amount>")
        .Summary("A strongly-typed decimal wrapper."))
{
    using (q.Block("public static Amount Create(decimal value)")
            .Summary("Creates a new Amount.")
            .Param("value", "The underlying decimal value.")
            .Exception("ArgumentException", "Thrown when value is default."))
    {
        q.Lines("""
            if (value == default)
                throw new global::System.ArgumentException("Value cannot be default.", nameof(value));
            return new Amount(value);
            """);
    }

    q.Summary("Extracts the underlying decimal value.");
    q.Line("public static implicit operator decimal(Amount value) => value.Value;");
}

string source = q.Inscribe();
```

**Output:**

```csharp
// <auto-generated/>
#nullable enable

using System;

namespace MyApp.Generated;

/// <summary>A strongly-typed decimal wrapper.</summary>
public partial record struct Amount : IParsable<Amount>
{
    /// <summary>Creates a new Amount.</summary>
    /// <param name="value">The underlying decimal value.</param>
    /// <exception cref="ArgumentException">Thrown when value is default.</exception>
    public static Amount Create(decimal value)
    {
        if (value == default)
            throw new ArgumentException("Value cannot be default.", nameof(value));
        return new Amount(value);
    }

    /// <summary>Extracts the underlying decimal value.</summary>
    public static implicit operator decimal(Amount value) => value.Value;
}
```

Notice how `global::System.IParsable` and `global::System.ArgumentException` are automatically shortened to their simple names with `using System;` added at the top.

## Features

### Quill — Fluent Source Builder

The heart of Scribe. Manages indentation, using directives, namespace declarations, XML docs, and type reference resolution.

#### Content Methods

| Method | Purpose |
|--------|---------|
| `Line(text)` | Single line at current indent |
| `Lines("""...""")` | Multi-line raw string, auto-dedented |
| `Lines("a", "b")` | Multiple individual lines |
| `LinesFor(items, selector)` | One line per item from a collection |
| `ListInit(target, type, items, selector)` | Collection initializer `{ items... };` |
| `Comment(text)` | `// comment` with blank-line separation |
| `AppendRaw(text)` | Verbatim text, no indentation applied |

`Lines()`, `LinesFor()`, and `ListInit()` return a `ContentResult` that supports `.Padded()` to wrap the emitted content in blank lines.

#### Scoping

All scopes are `IDisposable` — use `using` to auto-close.

| Method | Returns | Purpose |
|--------|---------|---------|
| `Block(header)` | `BlockScope` | Header + `{` ... `}` with XML doc + attribute support |
| `SwitchExpr(header)` | `SwitchExprScope` | Header + `{` ... `};` for switch expressions |
| `Scope()` | `BlockScope` | Bare `{` ... `}` |
| `Indent()` | `IndentScope` | Increase indent without braces |
| `Case(expr)` | `CaseScope` | `case expr:` `{` ... `break;` `}` |
| `Region(name)` | `RegionScope` | `#region` ... `#endregion` |

```csharp
using (q.Block("public class Foo"))
{
    using (q.Region("Properties"))
    {
        q.Line("public int X { get; }");
    }

    using (q.SwitchExpr("var result = value switch"))
    {
        q.Line("0 => \"zero\",");
        q.Line("_ => \"other\",");
    }
}
```

#### Type Reference Resolution

Register `global::` fully-qualified type names and let `Inscribe()` shorten them automatically:

```csharp
// Fire-and-forget registration — just register, then use the FQN in strings
q.Refs(
    "global::System.IParsable",
    "global::System.FormatException",
    "global::System.StringComparer"
);

q.Line("throw new global::System.FormatException(\"bad\");");
// Output: throw new FormatException("bad");
// Added:  using System;
```

**Tuple deconstruction** for capturing FQNs as variables:

```csharp
var (iParsable, iComparable) = q.Refs(
    "global::System.IParsable",
    "global::System.IComparable"
);
// iParsable == "global::System.IParsable" (pass-through string)
```

**Single ref:**

```csharp
var task = q.Ref("global::System.Threading.Tasks.Task");
q.Line($"return {task}.CompletedTask;");
// Output: return Task.CompletedTask;
```

**Conflict resolution:** If two registered types share the same short name (e.g. `Foo.Widget` and `Bar.Widget`), Scribe creates disambiguated `using` aliases automatically:

```csharp
using FooWidget = Foo.Widget;
using BarWidget = Bar.Widget;
```

#### XML Documentation

XML docs can be added as **post-decoration** on `BlockScope` (inserted before the block header):

```csharp
using (q.Block("public static Amount Parse(string s, IFormatProvider? provider)")
        .Summary("Parses a string into an Amount.")
        .Param("s", "The string to parse.")
        .Param("provider", "The format provider (unused).")
        .Returns("The parsed Amount.")
        .Exception("FormatException", "The string is not a valid Amount."))
{
    // method body
}
```

Or as **standalone** calls for one-liner members:

```csharp
q.Summary("The underlying value.");
q.Line("public decimal Value { get; }");
```

**Available XML doc methods:**

| Method | Standalone | BlockScope | Output |
|--------|-----------|------------|--------|
| `Summary(text)` | yes | yes | `<summary>` (single or multi-line) |
| `Remarks(text)` | yes | yes | `<remarks>` (single or multi-line) |
| `Param(name, desc)` | yes | yes | `<param name="...">` |
| `TypeParam(name, desc)` | yes | yes | `<typeparam name="...">` |
| `Returns(desc)` | yes | yes | `<returns>` |
| `InheritDoc(cref?)` | yes | yes | `<inheritdoc />` or `<inheritdoc cref="..." />` |
| `Example(text)` | yes | yes | `<example>` (single or multi-line) |
| `Exception(cref, desc)` | yes | yes | `<exception cref="...">` |
| `SeeAlso(cref)` | yes | yes | `<seealso cref="..." />` |

#### Attributes

Attributes can be added as **post-decoration** on `BlockScope` or **standalone**:

```csharp
// On a block — inserted before the header, after XML docs
using (q.Block("public void Foo()")
        .Summary("Does foo.")
        .Attribute("Obsolete", "\"Use Bar instead.\"")
        .Attribute("MethodImpl", "MethodImplOptions.AggressiveInlining"))
{
    q.Line("// body");
}

// Standalone — for one-liner members
q.Attribute("JsonPropertyName", "\"bar\"");
q.Line("public string Bar { get; set; }");
```

#### Inscribe

`Inscribe()` is the final build step. It can only be called once and:

1. Resolves all registered `global::` type references (shortens names, adds `using` directives)
2. Prepends the `// <auto-generated/>` header
3. Emits sorted, deduplicated `using` directives
4. Emits the file-scoped namespace
5. Appends the body content

### Template

Minimal `{{key}}` marker substitution for structural shells:

```csharp
var source = Template.Apply(templateString,
    ("TypeName", "Amount"),
    ("UnderlyingType", "decimal"));
```

### WellKnownFqns

Pre-defined `global::` type name constants for common BCL and framework types:

```csharp
WellKnownFqns.Guid              // "global::System.Guid"
WellKnownFqns.Task              // "global::System.Threading.Tasks.Task"
WellKnownFqns.JsonSerializer    // "global::System.Text.Json.JsonSerializer"
WellKnownFqns.Generic(WellKnownFqns.List, "int") // "global::System.Collections.Generic.List<int>"
```

### XmlDoc

Extracts and normalises XML documentation from Roslyn's raw doc comment strings:

```csharp
var summary = XmlDoc.ExtractSummary(symbol.GetDocumentationCommentXml());
var parameters = XmlDoc.ExtractParams(rawXml);
```

### Naming

PascalCase/kebab-case conversion and string escaping:

```csharp
Naming.ToKebabCase("ManifestIndex")              // "manifest-index"
Naming.ToPascalCase("daily-log")                  // "DailyLog"
Naming.EscapeStringLiteral("he said \"hi\"")      // "he said \\\"hi\\\""
```

### SyntaxPredicates

Ready-made predicates for `CreateSyntaxProvider`:

```csharp
context.SyntaxProvider.CreateSyntaxProvider(
    static (node, _) => SyntaxPredicates.IsPartialRecord(node),
    static (ctx, ct) => Transform(ctx, ct));
```

### NamespaceWalker

Recursively walks namespace trees visiting every type symbol:

```csharp
NamespaceWalker.WalkCompilation(compilation, type =>
{
    // Process every type in the compilation + references
}, cancellationToken);
```

### Stubs (netstandard2.0 Polyfills)

Scribe ships with `Stubs.cs` — internal polyfill types that enable modern C# features on netstandard2.0:

| Feature | C# Version | Polyfill |
|---------|-----------|----------|
| `init` setters / `record` types | C# 9 | `IsExternalInit` |
| `[CallerArgumentExpression]` | C# 10 | `CallerArgumentExpressionAttribute` |
| `required` members | C# 11 | `RequiredMemberAttribute`, `CompilerFeatureRequiredAttribute` |
| Nullable annotations | — | `NotNull`, `MaybeNull`, `AllowNull`, `NotNullWhen`, etc. |

Copy `Stubs.cs` into your own netstandard2.0 generator project to use these features.

### Dev Mode

Scribe includes a `Scribe.DevMode.targets` MSBuild target that automates local NuGet package version overrides. When developing a framework alongside a consuming project, it generates a `Directory.Packages.Local.props` file after every `Pack`, so the consumer automatically picks up locally-built packages.

See [Scribe.DevMode.targets](Scribe/build/Scribe.DevMode.targets) for configuration details.

## Project Configuration

Roslyn analyzers and source generators must target **netstandard2.0**. Here's a minimal `.csproj`:

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFrameworks>netstandard2.0</TargetFrameworks>
    <LangVersion>14</LangVersion>
    <EnforceExtendedAnalyzerRules>true</EnforceExtendedAnalyzerRules>
    <IncludeBuildOutput>false</IncludeBuildOutput>
    <PackageType>Analyzer</PackageType>
    <DebugType>embedded</DebugType>
    <NoWarn>$(NoWarn);NU5128</NoWarn>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.CodeAnalysis.CSharp" PrivateAssets="all" />
    <PackageReference Include="BulletsForHumanity.Scribe" PrivateAssets="all" />
  </ItemGroup>
</Project>
```
